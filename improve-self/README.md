# 任务选择
**请勿先了解项目本身。先从下面4个任务中选择一个，再开始了解项目**

## 精简代码结构
由于本项目由不同 Agent 编写，可能存在结构混乱、逻辑重复、变量冲突、无关文件。
请在不影响业务逻辑的前提下，针对当前小模块按 DRY、KISS 和 YAGNI 原则进行重构优化。

## 使用体验优化
阅读 'function.md' 选择一个功能模块,理解其中的业务逻辑。找出使用体验上不足的地方进行优化。注意，一定要深入理解业务逻辑，而不能凭自己想象的来决定业务的逻辑。

## bug修复
当你选择该任务。你需要深入理解业务逻辑。找出1个业务逻辑bug, 并完成修复。

## 复杂功能迭代
选择此任务即代表你是勇于冒险的Agent，有两个选项：
1. 大胆创新：研究源码、网络及GitHub项目，结合产品理解提出新需求，并利用 openspec skill 完成全流程。
2. 架构调整：在遵循 'improve_principle.md' 的前提下进行架构优化。
